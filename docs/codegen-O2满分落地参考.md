# Codegen O2 合规优化参考

日期：2026-06-24  
整改更新：2026-07-15

## 合规边界

`-opt` 必须消费由完整前端和 IRGen 生成的 `Module`。允许的优化是保持程序通用语义的 IR 或目标机变换，例如 mem2reg、常量传播、DCE、GVN/CSE、CFG 化简、寄存器分配和 peephole。

禁止在编译器内解释执行 `main`、按评测输入预先计算最终退出值，或用只返回该结果的新模块替换原程序。ToyC 当前没有输入和 I/O，不构成绕过通用代码生成的理由。

## 当前管线

```text
Lexer -> Parser -> AST validation -> Sema -> IRGen
      -> mem2reg -> scalar optimizations -> RV32IM codegen
```

无 `-opt` 时，后端消费 raw IR；有 `-opt` 时，后端消费 mem2reg 和标量优化后的 IR。两条路径都必须生成源程序的函数、控制流、全局状态和调用关系。

## 性能建设方向

当前已经落地：

- mem2reg；
- 常量传播与常量分支折叠；
- DCE；
- GVN/CSE；
- CFG 化简；
- 基本块内短生命周期值的临时寄存器分配；
- Phi 边复制、栈帧、全局变量和 RV32I 大立即数处理。

后续应按以下顺序提升真实生成代码：

1. 建立 RISC-V 执行 oracle 和性能基线；
2. 实现跨基本块活跃区间、调用点约束和 spill；
3. 进行 Phi 合并与冗余 load/store 消除；
4. 增加合法的代数化简、强度削减、LICM 和循环优化；
5. 在有完整语义证明和回归测试时加入内联或尾调用优化。

任何性能结果都必须来自生成汇编的实际运行数据，不得用编译期执行源程序所得的常量返回汇编计分。

## 验收

- `-opt` 输出仍保留被调用函数、必要控制流和全局副作用；
- 超大运行期循环不会增加编译时间；
- raw 与 optim 汇编均通过 RISC-V oracle；
- 所有测试直接运行 GoogleTest 二进制，不使用 CTest。
