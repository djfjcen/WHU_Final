## 总目标

把当前管线：

```text
stdin -> Lexer -> Parser -> AST validation -> Sema -> IRGen -> IRPrinter
```

扩展为：

```text
stdin -> Lexer -> Parser -> AST validation -> Sema -> IRGen -> Codegen -> stdout RV32IM asm
```

并在 `-opt` 路径预留后续：

```text
Sema/IRGen -> mem2reg -> run_optim -> deSSA/codegen fallback
Sema/AST or IR -> whole-program const evaluation -> constant-return asm
```

第一阶段必须产生真实汇编并覆盖完整 ToyC 语义；整程序求值只在真实 codegen 可用之后接入。

## 设计边界

### 先做真实 codegen

真实 codegen 指从 `toyc::Module` 翻译为 RV32IM 汇编，至少覆盖：

- 全局变量和全局常量。
- 函数定义。
- 参数传递。
- 局部变量 `alloca/load/store`。
- 算术、比较、一元运算。
- `if`、`while`、`break`、`continue` 已经在 IR 中表现为基本块和跳转。
- 函数调用和递归。
- void 函数。
- 多个 `ret`。

真实 codegen 分两层推进：

| 层 | 输入 | 目的 |
|---|---|---|
| raw codegen | IRGen 非 SSA IR | 先覆盖完整 ToyC 功能 |
| optimized codegen | mem2reg + run_optim 后的 SSA IR | 支持 `-opt` fallback 和性能路径 |

raw codegen 可先完成；optimized codegen 需要 phi/deSSA，不应长期缺失。

### 后做 const 求值

const/整程序求值是 `-opt` 的性能捷径，不是第一阶段 codegen 的替代品。它必须满足：

- 有预算限制。
- 求值失败 fallback 到真实 codegen。
- 不猜值。
- 不改变全局写、函数调用、副作用顺序。

本 plan 的 P8 只预留求值接入点，不在 P0-P7 中实现。

## 公共接口

`include/toyc/codegen.h`：

```cpp
#pragma once

#include <iosfwd>

namespace toyc {

class DiagnosticEngine;
class Module;

struct CodegenOptions {
    bool opt_mode = false;
    bool emit_exit_syscall = false;
    bool allow_pseudo = true;
};

bool emit_riscv(const Module& module, const CodegenOptions& options,
                DiagnosticEngine& diagnostics, std::ostream& out);

}  // namespace toyc
```

Driver 接入点：

```cpp
toyc::SemaResult sema = toyc::analyze(*unit, diagnostics);
std::unique_ptr<toyc::Module> ir = toyc::generate(*unit, sema, diagnostics);
toyc::CodegenOptions cg_options;
cg_options.opt_mode = options.opt_mode;
return toyc::emit_riscv(*ir, cg_options, diagnostics, std::cout) ? 0 : 1;
```

| C | Frame/value layout | `frame.cpp`, `lowering.cpp` | A/B | alloca/global/value 的机器位置 |
| D | Instruction lowering | `lowering.cpp` | B/C | 算术、比较、load/store、branch、ret |
| E | ABI/function calls | `lowering.cpp`, `frame.cpp` | C/D | 参数、call、ra、返回值、递归 |
| F | Test harness | `test/codegen/*`, `Justfile` | A/B | GoogleTest asm snapshot 和可选 oracle 脚本 |

## P2: Frame 和非 SSA 值位置

目标：当前 IRGen 产物是非 SSA `alloca/load/store`。先建立正确、清楚的 frame/value 规则。

### 核心设计

第一版可采用“真实但保守”的 value strategy：

- 每个 `AllocaInst` 结果对应一个 i32 frame slot。
- 每个产生结果的非内存指令也可有一个 temporary result slot。
- lowering 每条指令时用 `t0/t1/t2` 做短生命周期临时寄存器。
- `ConstantInt` 直接 materialize。
- `GlobalAddr` materialize 为地址。
- `Param` 在函数入口从 ABI 位置保存到 IRGen 已创建的参数 alloca slot。

这仍是真实 codegen，因为输出的是完整可运行汇编；只是寄存器分配保守。P7 会升级。

### FrameLayout

需要记录：

| 内容 | 说明 |
|---|---|
| `ra` 保存槽 | 非 leaf 或统一保存 |
| `s0/fp` 保存槽 | 第一版可统一使用 frame pointer |
| alloca slots | `AllocaInst* -> offset` |
| temp result slots | `Instruction* -> offset` |
| outgoing arg area | 参数超过 8 或 call 需要 |
| frame size | 16 字节对齐 |

offset 约定必须统一。建议以 `sp` 为基址，frame 建立后所有 slot 使用正 offset：

```asm
addi sp, sp, -FRAME_SIZE
sw ra, RA_OFFSET(sp)
...
lw ra, RA_OFFSET(sp)
addi sp, sp, FRAME_SIZE
ret
```

### ValueMaterializer

建议提供内部 helper：

- `load_i32(Value* value, RvReg dst)`
- `store_i32(RvReg src, Value* destination_ptr)`
- `load_address(Value* ptr, RvReg dst)`
- `spill_result(const Instruction& inst, RvReg src)`
- `move_to_a0(Value* value)`

### 验收

- [ ] `return const`
- [ ] `return local var`
- [ ] `local arithmetic`
- [ ] `global var read/write`
- [ ] `global const read`
- [ ] `negative constants`
- [ ] large constants through `li` or expansion

## P3: Instruction lowering

目标：覆盖当前 `Opcode` 中真实 codegen 必需的指令。

### 算术和一元

- [ ] `Add`
- [ ] `Sub`
- [ ] `Mul`
- [ ] `Sdiv`
- [ ] `Srem`
- [ ] `Neg`
- [ ] `Shl`
- [ ] `Shr`

注意：`Shr` 当前 IR 没区分逻辑/算术。ToyC int 是有符号；若该指令由后续优化产生，默认按算术右移 `srai`，并在文档中保持一致。

### 比较

比较结果必须物化为 0/1：

- [ ] `ICmpSlt`: `slt`
- [ ] `ICmpSgt`: swapped `slt`
- [ ] `ICmpSle`: `slt tmp, rhs, lhs; xori dst, tmp, 1`
- [ ] `ICmpSge`: `slt tmp, lhs, rhs; xori dst, tmp, 1`
- [ ] `ICmpEq`: `xor tmp; sltiu dst, tmp, 1`
- [ ] `ICmpNe`: `xor tmp; sltu dst, x0, tmp`

### 内存

- [ ] `Alloca`: 不发指令，只占 frame slot。
- [ ] `Load`: 从 stack slot 或 global address 读取。
- [ ] `Store`: 写 stack slot 或 global address。

`store` 操作数顺序是 `store <ptr>, <value>`。

### 控制流

- [ ] 每个 IR block 输出唯一 asm label。
- [ ] `Br`: 跳目标 label。
- [ ] `CondBr`: `bne cond, x0, true_label` + `j false_label`。
- [ ] fallthrough 优化可以后做，第一版保持显式跳转。

### 返回

- [ ] `Ret(value)`: 把 value 放到 `a0`，跳统一 epilogue。
- [ ] `Ret()`: 跳统一 epilogue。
- [ ] 每个函数一个 exit label。

### Phi

raw IRGen 不产生 `Phi`，但 `mem2reg` 和 `run_optim` 已经会产生 `Phi`。P3 只承诺 raw codegen；遇到 `Phi` 可以诊断失败：

```text
codegen does not support phi before deSSA
```

但 P3 完成后要在计划中登记：P3 支持非 SSA IR；SSA/phi 支持属于 P6 deSSA，且是 `-opt` codegen 的前置条件。

### 验收样例

- [ ] `if / else`
- [ ] `while`
- [ ] `break / continue`
- [ ] `&& / ||`，通过 IRGen 的短路 alloca 形态
- [ ] relational chains
- [ ] nested blocks with shadowed names，实际由 IR slot 区分

## P4: ABI 和函数调用

目标：支持多函数、递归、void 函数和参数传递。

### 参数

- [ ] 函数入口读取 `a0-a7`。
- [ ] 超过 8 个参数时按 RISC-V ABI 从调用者栈区读取。
- [ ] IRGen 已经在 entry 里 `store param_slot, %arg.N`，lowering `Param` 时要能 materialize 参数值。

### 调用

- [ ] 计算每个实参。
- [ ] 前 8 个放入 `a0-a7`。
- [ ] 超过 8 个写入 outgoing arg area。
- [ ] 保存 call 后仍需使用的 caller-saved 值。第一版若所有中间结果已 spill 到 stack，可只处理临时寄存器生命周期，简单很多。
- [ ] `call` 或 `jal ra, callee`。
- [ ] call 有返回值时保存 `a0` 到该 `CallInst` 结果位置。
- [ ] void call 不保存结果。

### Prologue/Epilogue

第一版允许统一保存 `ra` 和 `s0`，但 P7 要优化 leaf 函数。

### 验收样例

- [ ] `add(a,b)`
- [ ] nested call `f(g(1), h(2))`
- [ ] recursive factorial
- [ ] recursive gcd
- [ ] void function mutating global var
- [ ] function with more than 8 args

## P6: SSA / Phi / deSSA 接入

目标：让 codegen 能消费 `mem2reg` 和 `run_optim` 后的 IR。这是 `-opt` 正式走优化 IR 的前置条件。

### 输入形态

- `-dump-ir -mem2reg-only` 已可产生 SSA IR。
- `-dump-ir -opt` 已可产生优化后 SSA IR。
- 优化后 IR 可能含 `Phi`，也可能已被 CFS 化简掉。
- 当前 optim pass 包含 constprop、DCE、GVN、CFS；不要在 codegen 里重复实现这些 pass。

### deSSA 策略

建议第一版使用 codegen 前 deSSA：

- [ ] 遍历每个 block 开头的 phi。
- [ ] 对每条 predecessor edge 收集 copy：`phi_result <- incoming_value`。
- [ ] 在 predecessor terminator 前插入或发射 copy。
- [ ] parallel copy 不能顺序覆盖旧值；遇到环时使用临时寄存器或临时 spill slot。
- [ ] phi result 进入普通 value location 表，后续 lowering 不再特殊看 phi。
- [ ] deSSA 后 codegen 不应再看到 `Phi`。

如果不想修改 IR，可在 lowering 阶段按 edge 发射 copy，但仍必须处理 parallel copy。

### 验收样例

- [ ] `-mem2reg-only` if/else phi。
- [ ] `-mem2reg-only` while loop header phi。
- [ ] `-opt` constant branch folded 后无 phi。
- [ ] `-opt` CSE 后共享 value。
- [ ] `-opt` sample.tc 可生成 asm。

## P7: 从保守栈式到寄存器化 fallback

目标：真实 codegen 可用后，避免 `-opt` fallback 性能过弱。

### 必做优化

- [ ] Leaf 函数不保存 `ra`。
- [ ] 无 frame slot 时不调整 `sp`。
- [ ] 不需要 frame pointer 时省略 `s0`。
- [ ] fallthrough block 删除多余 `j`。
- [ ] `cond_br` 直接使用比较分支：能从 `icmp + cond_br` 合并时不要先物化 0/1。
- [ ] 小立即数使用 I 型指令。
- [ ] 删除 `mv x, x`、`addi x, x, 0`。

### 简单寄存器分配

在 mem2reg/optim 已实现的前提下，P7 可以先做 local linear scan：

- 基本块内计算 use positions。
- 只把纯计算和 load 结果分配到临时寄存器。
- call 边界 spill live caller-saved。
- alloca/global memory 语义仍保持。

后续再升级到函数级 allocation、更完整的 spill 和 copy coalescing。

### Phi/deSSA

P6 已提供第一版 phi/deSSA。后续增强需要：

- [ ] CFG 前驱/后继完整维护或现场推导。
- [ ] critical edge splitting。
- [ ] phi lowering 为 parallel copy。
- [ ] copy coalescing。

## P8: `-opt` const/整程序求值接入

目标：在真实 codegen 存在后，给 `-opt` 增加满分性能机会。

### 接入原则

```text
if options.opt_mode:
    try whole-program evaluation with budget
    if success:
        emit constant-return asm
    else:
        emit real codegen fallback
else:
    emit real codegen
```

这里的 `real codegen fallback` 应优先是 P6/P7 后的优化 IR codegen，而不是 raw 栈式 codegen。

### 约束

- [ ] 求值器必须严格模拟 ToyC 可观察语义。
- [ ] 全局 var 写入和 void 调用不能跳过。
- [ ] 递归和循环必须有预算。
- [ ] 求值失败必须 fallback。
- [ ] 成功结果必须经过 oracle 用例验证。

### 文件建议

```text
include/toyc/whole_program_eval.h
src/opt/whole_program_eval.cpp
```

不要放进 `src/codegen/`，避免后端和高级优化粘死。

## 风险记录

| 风险 | 对策 |
|---|---|
| 评测 main 退出机制未知 | `CodegenOptions::emit_exit_syscall` 保留开关，集中在 epilogue |
| RISC-V 伪指令支持未知 | `allow_pseudo` 保留开关，常量/地址装载集中封装 |
| 当前 BasicBlock `preds/succs` 不完整 | codegen 从 terminator 操作数推导 CFG |
| 栈 frame offset 混乱 | P2 单测锁定 frame layout |
| 调用覆盖 caller-saved 值 | 第一版全中间结果 spill，P6 再优化 |
| `Phi` 未支持 | P0-P5 明确只消费 raw 非 SSA IR，P6 必须补 deSSA 后才能声明 `-opt` codegen 完整 |
| const 求值过度承诺 | P8 设预算和 fallback，真实 codegen 先行 |

## Agent 执行规则

- 每个 agent 只改自己模块相关文件。
- 改 driver/CMake/Justfile 的 agent 必须跑 `just build` 和相关测试。
- 不允许添加 CTest。
- 不允许删除或改写无关文档。
- 新增测试必须是 GoogleTest binary 直接运行，或可选 shell oracle。
- 任何阶段如果需要改变 IR 语义，先更新 `docs/codegen-IR输入契约.md`。
