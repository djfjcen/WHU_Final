# 移除整程序求值的整改记录

日期：2026-07-15

## 问题

旧版驱动在 `-opt` 模式下、语义分析之后调用 ToyC 整程序解释器。求值成功时，它跳过原程序的 IRGen 和通用 codegen，只生成返回最终常量的 `main`。该行为不符合“完整编译为等价 RISC-V 汇编”的实验要求。

## 已删除内容

- `include/toyc/eval.h` 与 `src/eval/eval.cpp`；
- `evaluate_program`、`EvalBudget` 和 15 秒求值预算；
- 驱动中的最终返回值快路径；
- `toyc-eval-tests` 及对应 CMake 源文件；
- 为加速解释器而绕过构建类型、强制施加的 `-O2`；
- 将整程序解释执行描述为性能优化主路径的文档内容。

Git 历史未改写，可继续审计问题的引入和整改过程。

## 当前合规边界

正式编译必须经过 `Lexer -> Parser -> AST validation -> Sema -> IRGen -> RISC-V codegen`。`-opt` 只能在 IR 和目标机层执行保持通用语义的变换，不得执行源程序、求最终退出值或按评测结果重建程序。

## 防回归

- 驱动入口被抽为可直接测试的 `run_compiler`；
- 优化管线测试检查函数、调用、循环和全局访问仍出现在汇编中；
- 十亿次循环用例检查编译不会随运行期迭代次数增长；
- codegen 样例同时覆盖 raw 和 optim 管线；
- RISC-V oracle 用于验证实际汇编结果。
