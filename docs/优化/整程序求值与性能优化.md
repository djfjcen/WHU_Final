# ToyC 合规性能优化记录

整改日期：2026-07-15

## 设计边界

旧版本曾在 `-opt` 下解释执行整个 ToyC 程序，并把结果替换为常量返回汇编。该路径绕过 IRGen 和通用代码生成，已经删除。本文不再将其视为可接受的编译优化。

当前所有正式编译统一经过：

```text
Lexer -> Parser -> AST validation -> Sema -> IRGen
      -> optional mem2reg/run_optim -> RISC-V codegen
```

## 已实现的真实优化

- `mem2reg`：把可提升的局部栈槽转换为 SSA 值并生成 Phi；
- 常量传播：折叠局部常量表达式和常量条件；
- DCE：删除没有可观察效果的无用指令；
- GVN/CSE：在合法作用域内复用等价表达式；
- CFG simplify：删除不可达块、折叠分支并简化 Phi；
- 后端局部寄存器规划：让基本块内、不跨调用的短生命周期值使用 `t3` 至 `t6`；
- 目标机 lowering：处理函数调用、全局变量、Phi 边复制、栈帧和超出 12 位的立即数。

## 当前限制

后端仍以栈槽作为大多数 IR 值的保守存储，跨基本块值和循环携带值没有完整的全局寄存器分配，因此大型循环与 gcc `-O2` 仍有明显差距。后续优化必须在生成汇编上实现和测量，不能把运行期工作转移成编译器内对源程序的解释执行。

## 验证方式

- raw 与 optim 两条管线编译同一组 codegen 用例；
- 驱动级测试确认优化汇编保留函数、调用、循环和全局访问；
- 十亿次循环只编译不执行，用于防止重新引入编译期整程序执行；
- 使用 RISC-V oracle 比较生成汇编的退出值；
- 直接运行 GoogleTest 二进制，不使用 CTest。
